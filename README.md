# 🕌 Времена за Намаз (Prayer Times Ecosystem)

Цялостна мултиплатформена екосистема за прецизно изчисляване, визуализация и известяване на времената за ислямски молитви (намаз). Проектът се отличава с **пълна офлайн автономност**, **астрономическа прецизност** и интеграция между уеб, мобилни устройства, десктоп и специализиран хардуер (смарт часовник).

## <a id="about"></a> 🎯 За проекта

Основната цел на системата е да предостави точна, надеждна и независима платформа, която не изисква постоянна връзка с интернет (Offline-First подход). Вместо да изтегля готови таблици от сървър, системата генерира времената в реално време на самото устройство, използвайки сложни математически и астрономически алгоритми, базирани на географските координати на потребителя.

---

## <a id="features"></a> ✨ Основни функционалности

* **100% Офлайн работа:** Всяка платформа разполага със собствен автономен изчислителен модул.
* **Мултиезичност:** Поддръжка на Български, Английски, Турски и Арабски език.
* **Достъпност (Accessibility):** Вградени режими за висок контраст, далтонизъм, уголемен текст и гласово четене (Text-to-Speech).
* **Интелигентни известия:** Персонализирани нотификации и живо отброяване (countdown) до следващата молитва.
* **Кибла компас:** Определяне на посоката и разстоянието до Мека.
* **Административен панел и Е-магазин:** Интегрирана платформа за управление на потребители и поръчки на специализирания смарт часовник.

## <a id="structure"></a> 📁 Структура на хранилището

Проектът е разделен на 4 основни директории (модули), всеки от които функционира независимо:

* `Web-PWA/` - Уеб платформа, PWA, Е-магазин, REST API и Админ панел
* `Mobile-App/` - Мобилно приложение за Android & iOS (React Native)
* `Desktop-App/` - Десктоп приложение за Windows работещо във фонов режим
* `Smartwatch/` - C/C++ фърмуер за специализиран водоустойчив смарт часовник

## <a id="core"></a> 🧮 Астрономическо ядро и изчисления

Най-голямото техническо предизвикателство в проекта е създаването на унифициран изчислителен модел. Алгоритъмът се базира на алгоритъма на Жан Меюс (Jean Meeus).

![Изображение 4. Схема на алгоритъма](assets/4_algorithm_schema.jpg)

Процесът включва:
1.  **Сферична тригонометрия:** Изчисляване на Юлианския ден, слънчевата деклинация ($\delta$) и Уравнението на времето. Използва се основната формула за Часовия ъгъл ($H$) за намиране на моментите, в които Слънцето достига специфични ъгли под хоризонта (напр. $-18^\circ$ за Фаджр и $-17^\circ$ за Иша).
2.  **Динамични корекции за реални условия:**
    * *Атмосферна рефракция:* Коригиране на оптичното огъване на светлината, отчитайки локалната температура и атмосферно налягане.
    * *Сезонни вариации:* Синусоидална функция за компенсиране на микро-отклоненията през годината.
3.  **Изчисление на Аср:** Времето се определя от момента, в който дължината на сянката на обект е равна на дължината на самия обект плюс дължината на сянката му по пладне.

![Изображение 3. Част от кода за алгоритъма](assets/3_algorithm_code.png)

4.  **Йерархия на точността:** За пълно съответствие с официалния календар за България се прилага пространствена интерполация спрямо референтни градове и се добавят микро-корекции чрез JSON таблици. При евентуални бъдещи разминавания, Python скрипт (`py.py`) преизчислява и калибрира данните.

## <a id="tech"></a> 🛠 Технологичен стек

Архитектурата следва стриктен 3-слоен модел: **Frontend → REST API → JSON/SQL хранилище**.

* **Мобилни приложения:** React Native, JavaScript
* **Уеб платформа & PWA:** HTML, CSS, JavaScript, PHP, Service Workers
* **Десктоп приложение:** C#, SQL
* **Смарт часовник (Ниско ниво):** C, C++ (Оптимизация за QMI8658 6-Axis IMU и пестене на батерия)
* **Бекенд & API:** PHP, Python, SQL, RESTful API (HTTP GET JSON)
* **Тестване и DevOps:** Git, GitHub Actions, Jest (Unit Testing)

![Изображение 8. Изглед на десктоп приложението](assets/8_desktop_app.jpg)

## <a id="testing"></a> 🧪 Тестване и CI/CD

За да се гарантира абсолютната точност на бизнес логиката, проектът използва съвременни DevOps практики:
* **Unit Тестове (Jest):** Файлове като `CalculatorUnitTest.test.js` проверяват правилното изчисляване на времената, хронологичния им ред и прилагането на локалните отмествания чрез симулиране (mocking) на външни зависимости.
* **GitHub Actions:** При всяко обновяване на кода (`git push`), виртуална среда автоматично изпълнява `npm test`, предотвратявайки човешки грешки и регресии в математическите алгоритми.

![Изображение 2. Unit тестове чрез GitHub Actions](assets/2_github_actions.png)
